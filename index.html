Math Functions in PostgreSQL

PostgreSQL provides several useful mathematical functions for calculations.

ABS()

ABS() returns the absolute value of a number â€” its distance from zero.

The absolute value of a number is its distance from zero on a number line, regardless of direction. This means that the absolute value of both 3 and -3 is 3 as each is 3 units away from zero.

NOTE: The absolute value is always positive, even is the original number is negative.

Example: SELECT ABS(-3); -- returns 3

CEIL()

CEIL() This function rounds a number up to the nearest integer.

It essentially returns the smallest integer that is greater than or equal to the input number, and it always chooses the integer that is on the positive side of the decimal value.

Unlike the FLOOR() function, which rounds down, CEIL() always rounds towards positive infinity

The output of CEIL() is always an integer, even if the input is a floating point number

Example: SELECT CEIL(4.2); -- returns 5

FLOOR()

FLOOR() This function rounds a number down to the nearest integer.

It takes a real number as the input and returns the largest integer less than or equal to that number.

It essentially rounds down to the nearest whole number.

Example: SELECT FLOOR(4.8); -- returns 4

Example: SELECT FLOOR(8.8345); -- returns 8

ROUND()

ROUND() This function rounds a number to the specified decimal places.

This function is essential for data manipulation when precision needs to be controlled in calculations or data presentation. It rounds up any portion of its first argument that is smaller that half a unit of the least significant digit or time unit within the precision of the second argument, but any portion of the first argument that is equal to or greater that half a unit is rounded down

Example: SELECT ROUND(4.567, 2); -- returns 4.57

Example: SELECT ROUND(5.7867, 2); -- returns 5.79

POWER()

POWER() This function is used to return the value of one number raised to the power of another number.

Some uses include:

  1. Exponential calculations: where exponentiation is involved.
  2. Mathematical operations: useful in various formulas that require raising a number to a power
  3. Scientific research: employed in scientific computations involving growth rates, decay, and other exponential models.
  4. Financial modeling: to calculate the compound interest and other financial scenarios requiring exponentiation.

Example: SELECT POWER(2, 3); -- returns 8

Example: SELECT POWER(6, 3); -- returns 216

SQRT()

SQRT() This function allows you to calculate the square root of a number

It returns the square root of a number

Example: SELECT SQRT(9); -- returns 3

MOD()

MOD()This function allows you to perform a modulo operation, returning the remainder after dividing the first argument by the second argument.

In this syntax: dividend: the dividend is the number that you want to divide Divisor: Its the number by which you want to divide the dividend

Example: SELECT MOD(10, 3); -- returns 1

Example: SELECT MOD(124, 5); -- returns 4

RANDOM()

RANDOM() This function returns a random float value between 0.0 and 1.0.

Example: SELECT RANDOM(); -- returns something like 0.7652

TRUNC()

TRUNC() truncates a number to a whole number or a specified number of decimal places.

Example: SELECT TRUNC(4.987, 2); -- returns 4.98
